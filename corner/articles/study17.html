$\epsilon$-greedy

$Q$ 值可以衡量某一状态 $s$ 下动作 $a$ 的优势： $$Q_{\pi}(s,a)=\mathbb{E}_{\pi}[R_{t+1}+\gamma V_{\pi}(S_{t+1})\mid S_t=s,A_t=a]$$ 有了 $Q$ 值对策略进行评估之后，只需要找到能提升 $Q$ 值的方法就可以提升策略的效果，最简单的方法就是贪心算法： $$\pi^*(s)=\mathop{\arg\max}_{a'}Q_{\pi}(s,a')$$ 贪心算法一定会得到比之前更优的策略，因为： $$Q_{\pi^*}(s,a)=\max_{a'}Q_{\pi}(s,a')\geqslant Q_{\pi}(s,a)$$

设 $A$ 为动作空间，$s$ 为当前状态，$\epsilon$-greedy 其实就是以 $1-\epsilon$ 的概率贪心，否则在动作空间中随机选取动作：

        \begin{algorithm}
            \caption{$\epsilon$-greedy}
            \begin{algorithmic}
                \IF{${\rm Uniform}(0,1)<\epsilon$}
                    \STATE\RETURN a random action from $A$
                \ELSE
                    \STATE\RETURN $\arg\max_{a'\in A}Q(s,a')$
                \ENDIF
            \end{algorithmic}
        \end{algorithm}

$Q$-Learning 算法

在状态 $s$ 下，$Q$-Learning 算法首先从 $Q$ 中用行为策略（例如 $\epsilon$-greedy）选取并执行动作 $a$，观察奖励 $r$ 和新的状态 $s'$，再更新 $Q$： \begin{align*} Q(s,a)=&\ Q(s,a)+\alpha(r+\gamma\max_{a'}Q(s',a')-Q(s,a))\\ =&\ (1-\alpha)Q(s,a)+\alpha(r+\gamma\max_{a'}Q(s',a')) \end{align*} 

        \begin{algorithm}
            \caption{$Q$-Learning}
            \begin{algorithmic}
                \REPEAT
                    \STATE Choose $a$ from $s$ using policy derived from $Q$ (e.g., $\epsilon$-greedy)
                    \STATE Take action $a$, observe reward $r$ and the next state $s'$
                    \STATE $Q(s,a)\gets Q(s,a)+\alpha(r+\gamma\max_{a'}Q(s',a')-Q(s,a))$
                    \STATE $s\gets s'$
                \UNTIL{max iterations}
            \end{algorithmic}
        \end{algorithm}

SARSA (State-Action-Reward-State-Action)

在状态 $s$ 下，SARSA 算法首先基于 $\epsilon$-greedy 选取并执行动作 $a$，观察奖励 $r$ 和新的状态 $s'$。在新的状态下同样基于 $\epsilon$-greedy 选择并执行动作 $a'$，然后更新 $Q$： \begin{align*} Q(s,a)=&\ Q(s,a)+\alpha(r+\gamma Q(s',a')-Q(s,t))\\ =&\ (1-\alpha)Q(s,a)+\alpha(r+\gamma Q(s',a')) \end{align*} 

        \begin{algorithm}
            \caption{SARSA}
            \begin{algorithmic}
                \STATE Choose $a$ from $s$ using policy derived from $Q$ (e.g., $\epsilon$-greedy)
                \REPEAT
                    \STATE Take action $a$, observe reward $r$ and the next state $s'$
                    \STATE Choose $a'$ from $s'$ using policy derived from $Q$ (e.g., $\epsilon$-greedy)
                    \STATE $Q(s,a)\gets Q(s,a)+\alpha(r+\gamma Q(s',a')-Q(s,a))$
                    \STATE $s\gets s'$
                    \STATE $a\gets a'$
                \UNTIL{max iterations}
            \end{algorithmic}
        \end{algorithm}

On-policy 和 Off-policy

On-policy 是同策略，即行为策略和目标策略相同，而 off-policy 是异策略，即行为策略和目标策略不同。行为策略 (behavior policy) 是采样数据并选取动作的策略，而目标策略 (target policy) 是使用这些数据来更新的策略。很容易看出，$Q$-Learning 算法是 off-policy 的，因为它的行为策略 ($\epsilon$-greedy) 和目标策略（贪心）不一致；而 SARSA 是 on-policy 的（行为策略和目标策略都是 $\epsilon$-greedy）。

DQN 算法

$Q$-Learning 算法不能处理连续状态 / 动作空间的情况，对于这种情况，可以使用函数来拟合 $Q$ 值，DQN 其实就是 $Q$-Learning 算法和神经网络的结合，它用一个神经网络 $Q_\phi$ 来拟合 $Q$ 函数，网络输入是状态 $s$ 和动作 $a$，输出是 $Q(s,a)$ 的近似值。根据 $Q$-Learning 算法的更新公式可知 $Q$ 网络的损失函数是： $$\mathcal{L}_\phi=\dfrac{1}{2}\left(Q_\phi(s,a)-\left(r+\gamma\max_{a'}Q_\phi(s',a')\right)\right)^2$$

Experience Replay（经验回放）

为了更好地将 $Q$-Learning 算法和神经网络结合，DQN 算法采用了经验回放方法，即维护一个回放缓存 (Replay Buffer)，将每次从环境中采样的 $(s,a,r,s')$ 存储到缓存中，训练 $Q$ 网络的时候再从缓存中随机采样若干数据来进行训练。

Target Network（目标网络）

DQN 算法的目标是让 $Q_\phi(s,a)$ 逼近 $r+\gamma\max_{a'}Q_\phi(s',a')$，由于目标函数本身就包含神经网络的输出，所以更新网络参数的同时目标函数也在改变，这会导致训练的不稳定。因此 DQN 算法采用了目标网络 $Q_{\phi'}$，用 $Q_{\phi'}$ 计算 TD 目标，每次更新的是 $Q_\phi$，而 $Q_{\phi'}$ 是定期基于 $Q_\phi$ 更新。

        \begin{algorithm}
            \caption{DQN}
            \begin{algorithmic}
                \STATE Initialize parameter $\phi$ of $Q$ network
                \STATE Initialize parameter $\phi'\gets\phi$ of target network
                \STATE Initialize replay buffer $R$
                \FOR{each episode}
                    \STATE Get the initial state $s_1$
                    \FOR{$t=1\to T$}
                        \STATE Choose $a_t$ from $s_t$ using policy derived from $Q_\phi$ (e.g., $\epsilon$-greedy)
                        \STATE Take action $a_t$, observe reward $r_t$ and next state $s_{t+1}$
                        \STATE Push $(s_t,a_t,r_t,s_{t+1})$ to replay buffer $R$
                        \IF{there is enough data in buffer $R$}
                            \STATE Sample $\{(s_i,a_i,r_i,s_{i+1})\}_{i=1}^N$ from buffer $R$
                            \STATE $\forall 1\leqslant i\leqslant N$, compute $y_i=r_i+\gamma\max_aQ_{\phi'}(s_{i+1},a)$ using target network
                            \STATE Compute loss $L=\frac{1}{N}\sum_{i=1}^N\left(y_i-Q_{\phi}(s_i,a_i)\right)^2$
                            \STATE Update $Q$ network: $\phi\gets\phi-\alpha\nabla_\phi L$
                            \STATE Update target network: $\phi'\gets\tau\phi+(1-\tau)\phi'$ periodically
                        \ENDIF
                        \STATE $s_t\gets s_{t+1}$
                    \ENDFOR
                \ENDFOR
            \end{algorithmic}
        \end{algorithm}

策略梯度 (Policy Gradient, PG)

$Q$-Learning 和 SARSA 都属于 Value-based 方法，即基于值（$Q$-table）的方法。但是强化学习的训练数据分布不稳定，因为 Value-based 的方法需要根据当前策略来估计，而策略是不断更新的，因此各个状态的访问频率存在很大的差别。

PG 属于 Policy-based 方法，即基于策略的方法。策略梯度的目标是直接优化策略 $\pi$，相当于找到一个函数 $F$，基于当前状态 $s$ 可以直接得到在各个动作上的概率分布，即 $P(a,s)=F(s)$，Policy-based 方法相比于 Value-based 方法的最大优势是适合解决具有高维或连续动作空间的问题。

PG 从初始状态开始，基于策略选取并执行动作，得到状态轨迹 $\tau=\{s_1,a_1,s_2,a_2,\cdots,s_T,a_T\}$，$\tau$ 在策略 $\pi_\theta$ 下发生的概率为（$\theta$ 为参数）： \begin{align*} p_\theta(\tau)=&\ p(s_1)p_{\theta}(a_1\mid s_1)p(r_1,s_2\mid s_1,a_1)p(a_2\mid s_2)p(r_2,s_3\mid s_2,a_2)\cdots\\ =&\ p(s_1)\prod_{t=1}^Tp_\theta(a_t\mid s_t)p(r_t,s_{t+1}\mid s_t,a_t) \end{align*} 所以可以根据采样得到回报 (Reward) 的期望 $\bar{R}_{\theta}=\sum_\tau R(\tau)p_{\theta}(\tau)$，目的显然是最大化这个期望值，可以根据梯度上升来最大化它： $$\nabla_\theta\bar{R}_\theta=\sum_\tau R(\tau)\nabla_\theta p_{\theta}(\tau)=\sum_\tau R(\tau)p_\theta(\tau)\dfrac{\nabla_\theta p_{\theta}(\tau)}{p_{\theta}(\tau)}=\sum_\tau R(\tau)p_\theta(\tau)\nabla_\theta\log p_{\theta}(\tau)$$ 而根据上面的式子可知： \begin{align*} \nabla_\theta\log p_\theta(\tau)=&\ \nabla_\theta\left(\log p(s_1)+\sum_{t=1}^T(\log p_\theta(a_t\mid s_t)+\log p(r_t,s_{t+1}\mid s_t,a_t))\right)\\ =&\ \sum_{t=1}^T\nabla_\theta\log p_\theta(a_t\mid s_t) \end{align*} 所以期望回报的梯度就可以写成： \begin{align*} \nabla_\theta\bar{R}_\theta=&\ \sum_\tau\left(R(\tau)p_\theta(\tau)\sum_{t=1}^T\nabla_\theta\log p_\theta(a_t\mid s_t)\right)\\ \approx&\ \dfrac{1}{N}\sum_{n=1}^N\sum_{t=1}^TR(\tau_n)\nabla_\theta\log p_\theta(a^n_t\mid s^n_t) \end{align*} 为了让不好的动作得到负反馈，可以让期望减去一个 baseline： $$\nabla_\theta\bar{R}_\theta\approx\dfrac{1}{N}\sum_{n=1}^N\sum_{t=1}^T(R(\tau_n)-b)\nabla_\theta\log p_\theta(a^n_t\mid s^n_t)$$ 现在就可以通过梯度上升来优化参数： $$\theta=\theta+\alpha\nabla_\theta\bar{R}_\theta$$

Actor-Critic

Actor-Critic 方法融合了 Value-based 和 Policy-based，其中 Actor 指的就是策略 $\pi_\theta$，而 Critic 指的是值函数 $Q$。Actor 需要采取让 Critic 满意的动作，Critic 需要准确估计出 Actor 所采取的动作的价值。假设 Actor 为 $\pi_\theta$，Critic 为 $Q_\phi$，其中 $\theta$ 和 $\phi$ 为参数，则基础的 Actor-Critic 算法就是在 Policy Gradient 上的延伸：

        \begin{algorithm}
            \caption{Actor-Critic}
            \begin{algorithmic}
                \STATE Sample $a\sim\pi_\theta(a\mid s)$
                \FOR{$t=1\to T$}
                    \STATE Take action $a$, observe reward $r$ and the next state $s'$
                    \STATE Sample the next action $a'\sim\pi_\theta(a'\mid s')$
                    \STATE $\theta\gets\theta+\alpha_{\theta} Q_w(s,a)\nabla_{\theta}\log\pi_{\theta}(a\mid s)$
                    \STATE Compute the TD error for current action-value: $\delta_t=r_t+\gamma Q_w(s',a')-Q_w(s,a)$
                    \STATE $\phi=\phi+\alpha_\phi\delta_t\nabla_\phi Q_\phi(s,a)$
                    \STATE $s\gets s'$
                    \STATE $a\gets a'$
                \ENDFOR
            \end{algorithmic}
        \end{algorithm}
