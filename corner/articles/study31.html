解决了什么问题

这篇文章的最后一个版本发布于 2022.11.03，那时 ChatGPT (2022.11.30) 还没有发布。而当时语言模型的训练基本上是以 Next Token Prediction (NTP) 为目标的有监督训练，这忽略了很多重要但潜在有用的信号，例如单元测试。为了解决这些局限性，作者提出了一种通过预训练语言模型和深度强化学习进行程序合成任务的新框架 CodeRL。

程序合成任务

假设 $D$ 是问题描述，$\hat{W}=(\hat{w}_1,\cdots,\hat{w}_T)\in\mathcal{V}^T$ 是模型输出的代码，在训练过程中，假设 $W=(w_1\cdots,w_T)$ 是 ground truth，$\theta$ 是模型参数，那么模型的目标是最小化： $$\mathcal{L}_{ce}(\theta)=-\sum_t\log p_\theta(w_t\mid w_{1:t-1},D)$$ 而推理过程中，模型自回归地生成代码，其中 $t$ 时刻输出的 token $\hat{w}_t$ 从分布 $p_\theta(\cdot\mid \hat{w}_{1:t-1},D)$ 中采样，然后使用测试用例来评估生成的代码，而 “测试样例” (example test cases) 一般会作为问题描述的一部分给出。

有监督学习的两个局限性

  • 仅以 teacher-forcing 的方式训练具有 NTP 目标的模型通常会在推理阶段累积误差。
  • 当时的模型没法充分利用单元测试，而单元测试直接通过程序的功能正确性来决定模型的性能。

CodeRL

为了解决这些问题，作者提出了 CodeRL 框架。具体地说，在训练过程中，作者将语言模型视作一个 Actor，且引入一个预训练的 Critic，它用来评估生成程序的功能正确性。而在推理过程中，作者引入了一个新的生成过程，该过程使用 Critical Sampling 策略，允许模型根据单元测试和 Critic 分数的反馈自动重新生成程序。

对于模型的 backbone，作者使用 CodeT5 作为 backbone，并做了以下调整：

  • 改进预训练数据：使用大规模 Github 代码数据集扩大 Python 预训练数据集，生成的 Python 数据集 (GCPY) 有 10.5B 个 tokens，是 CodeT5 使用的数据集 (CodeSearchNet) 的十倍大。
  • 改进预训练目标函数：引入了 NTP 的预训练任务。具体来说，作者对每个代码样本的 pivot 位置进行均匀采样，然后将 pivot 之前的内容传给编码器，剩余内容传给解码器。为了控制输入和输出序列的长度，pivot 被限制在原始序列的 10% 到 90% 之间。

将程序合成形式化为强化学习问题

将模型的可学习参数 $\theta$ 视作随机策略，在 $t$ 时刻，$\theta$ 采取动作，选择此时要输出的 token $w_t^s$，并且更新模型状态，在 episode 结束时，模型收到奖励值 $r$，这个奖励值和模型生成程序的正确性相关，所以模型的目标是： $$\min_\theta\ \mathcal{L}_{rl}(\theta)=\min_\theta\ -\mathbb{E}_{W^s\sim p_\theta}[r\left(W^s\right)]$$ 根据策略梯度理论： \begin{align*} \nabla_\theta\mathcal{L}_{rl}(\theta)\approx&\ -\mathbb{E}_{W^s\sim p_\theta}\left[r\left(W^s\right)\nabla_\theta\log p_\theta\left(W^s\mid D\right)\right]\\ \approx&\ -\mathbb{E}_{W^s\sim p_\theta}\left[r\left(W^s\right)\sum_t\nabla_\theta\log p_\theta(w^s_t\mid w^s_{1:t-1},D)\right] \end{align*}

使用测试用例定义回报 (return) 值

对于每个采样到的程序 $W^s$，回报 $r$ 定义为： $$r\left(W^s\right)=\left\lbrace\begin{array}{ll} -1.0&\text{if compile error}\\ -0.6&\text{if rumtime error}\\ -0.3&\text{if $W^s$ failed any test case}\\ +1.0&\text{if $W^s$ passed all test cases} \end{array}\right.$$ 为了减小方差，可以减去一个 baseline： $$\nabla_\theta\mathcal{L}_{rl}(\theta)=-\mathbb{E}_{W^s\sim p_\theta}\left[\left(r\left(W^s\right)-r\left(W^b\right)\right)\sum_t\nabla_\theta\log p_\theta(w^s_t\mid w^s_{1:t-1},D)\right]$$ 其中 $W^b$ 是通过贪心策略采样得到的程序，从而优于 baseline 的方法都会得到一个正的回报值，而劣于 baseline 的方法会得到一个负的回报值。

Intermediate Return by Critic as Error Predictor

上面的梯度估计仅基于解码过程结束时的最终回报，然而，程序通常遵循固定的语法规则，其中单个 token 可能会使程序出错，因此上面的式子可能过于严格，所以作者引入了参数为 $\phi$ 的 Critic，它接收问题描述 $D$ 和相应的程序 $W^s$ 为输入，输出 $W^s$ 的回报，所以 Critic 的目标函数为： $$\mathcal{L}_{\rm Critic}(\phi)=-\log p_\phi(u\mid W^s,D)$$ 其中 $u$ 为编译器基于测试用例给出的 ground truth 回报值。作者采用比 Actor 更小的 Transformer 作为 Critic 的基础架构，从 Critic 解码器获得的 tokens 的上下文隐状态 $(h_1,\cdots,h_T)$ 沿着序列长度维度进行最大池化： $$h^{\rm pool}={\rm Pooling}(h_1,\cdots,h_T)$$ Critic 的最终输出为： $$\hat{u}={\rm softmax}\left({\rm Linear}\left(h^{\rm pool}\right)\right)$$ 有了训练好的 Critic，可以使用概率分布 $\hat{v}_t={\rm softmax}({\rm Linear}(h_t))$ 来估计与 ground truth 测试输出相关的 $w_t^s$ 的 token 级别的值 $\hat{q}$（这里的 $h_t$ 没有经过池化），具体地： $$\hat{q}_\phi(w^s_t)=\hat{v}_t[u]$$ 其中 $\hat{v}[\cdot]$ 表示从四种可能的结果中得出特定结果的概率。从而，Actor 的目标函数为： $$\nabla_\theta\mathcal{L}_{rl}(\theta)=-\mathbb{E}_{W^s\sim p_\theta}\left[\left(r\left(W^s\right)-r\left(W^b\right)\right)\sum_t\hat{q}_\phi(w^s_t)\nabla_\theta\log p_\theta(w^s_t\mid w^s_{1:t-1},D)\right]$$
