LLaMa 1 (2023.02)

在大型语料库上训练的大语言模型展现出很强的小样本学习（Few-Shot Learning）能力，这在模型做到足够大时首次出现，所以很多工作都想把模型做大，都基于假设 “更多的参数带来更好的性能”。

但是 Hoffmann 等人提出（这篇文章还没看过，这两天看一下），对于给定的训练预算，最佳性能不是由最大的模型实现的，而是由较小模型在更大的数据集上实现的。不过他们忽略了推理预算（这在实际使用时至关重要）：给定目标性能，我们更希望模型的推理速度快，而不是训练得快。

LLaMa 1 的重点是在更大的数据集上训练更小的模型，以在给定的推理预算下实现尽可能好的性能。

LLaMa 1 模型架构

LLaMa 1 基于 Transformer，模型参数量在 7B 到 65B 不等，并做了以下调整：

  • 使用 Pre-normalization，即对每个 Transformer 子层的输入做归一化，而不是对输出做归一化；
  • 替换 ReLU 为 SwiGLU： $${\rm SwiGLU}(x,W,V,b,c,\beta)={\rm Swish}_{\beta}(xW+b)\odot(xV+c)$$ $${\rm Swish}_{\beta}(x)=x\sigma(\beta x)=\dfrac{x}{1+{\rm e}^{-\beta x}}$$
  • 替换绝对位置编码（Absolutely Positional Embedding）为旋转式位置编码（Rotary Positional Embedding, RoPE）。

LLaMa 1 的训练

LLaMa 1 使用 AdamW 优化器，超参数可以在原文找到，并且使用了一些技巧来加速训练：

  • 使用 Casual Multi-head Attention：不存储 attention 权重，且不计算被 mask 部分的 key/query score；
  • 通过保存一些计算量比较大的 activation 值（比如线性层的输出）来减少计算量。

65B 的 LLaMa 1 训练速度约为 380 tokens/s/GPU，所以在 2048 块 80G 的 A100 上，1.4T 的数据大约要训练 21 天（真有钱 QAQ）。有意思的是，文章还研究了能源和碳足迹（Carbon footprint），他们估算开发这些模型的耗电量大约为 2638 MWh（即 263.8 万度电），二氧化碳当量约 1015 tCO2eq。

LLaMa2 (2023.07)

当时的一些开源预训练大模型（例如 BLOOM，LLaMa 1 和 Falcon）已经可以和闭源预训练大模型（例如 GPT-3 和 Chinchilla）不相上下了，但是它们都不是闭源 “产品” 大模型（例如 ChatGPT，Bard 和 Claude）的合适替代品，因为这些 “产品” 都经过了严格的微调，以符合人类的偏好，这大大提高了它们的可用性和安全性。而 “微调” 步骤需要大量的计算成本和人工成本，而且往往不透明。

于是他们又发布了 LLaMa 2，一个预训练且微调过的大模型族，且有 LLaMa 2 和 LLaMa 2-Chat 两个版本：

  • LLaMa 2 是 LLaMa 1 的升级版。公开的 LLaMa 2 有 7B，13B 和 70B 三种大小，还有 34B 的但没公开；
  • LLaMa 2-Chat 是 LLaMa 2 微调后用于对话任务的产物，也有 7B、13B 和 70B 三种大小。

在一系列可用性和安全性的 benchmarks 上，LLaMa 2-Chat 比当时的开源大模型表现得更好，且和一些闭源 “产品” 不相上下。

预训练数据

LLaMa 2 的预训练语料库比 LLaMa 1 的更大（1.4T $\rightarrow$ 2T）且更干净（删除了包含大量个人信息的数据）。

代码填充 (Code Infilling)

微调出来的 Code LLaMa 只会做给定前面的代码预测后面的代码这种任务，还不会做已知上下文预测中间内容的任务（即代码补全），比如给定函数签名和具体实现生成 docstring。这种任务和大模型预测下一个 token 的任务模式不同，需要对训练模式改造。论文采用了 Casual Mask 的模式，即将训练序列中间的一部分移动到最后，自回归地预测这部分内容。具体来说，就是将训练序列分割成三部分：前缀、中间和后缀，分割位置利用均匀分布来确定。训练序列一半为前缀-后缀-中间（PSM）格式，另一半为后缀-前缀-中间（SPM）格式。

长上下文微调

LLaMa 2 的最长 token 数为 4096，对于代码生成任务来说是比较小的，为了适应 repository 级别的任务，需要对上下文长度进行扩充，Code LLaMa 在微调阶段将其扩充至原来的四倍，即 16384。实验表明这样训练出的 Code LLaMa 在 100K tokens 的任务下也表现得很稳定。

LLaMa3 (2024.04)

LLaMa 3 目前还没有具体的论文，只有官方的 blog。它有 8B 和 70B 两种大小。相比于 LLaMa 2，LLaMa 3 在模型架构上做了几个关键改进：

  • 使用了一个具有 128K tokens 的词汇表的 tokenizer；
  • 8B 和 70B 的 LLaMa 3 都使用了 GQA 来提升推理性能；
  • 采用 8192 tokens 序列进行训练，这样可以保证 Self-Attention 不会跨文档。

预训练数据

LLaMa 3 的训练语料库超过了 15T tokens，这比 LLaMa 2 的语料库大了七倍，并且其中的代码多了四倍。语料库中还有超过 5% 的高质量非英语（30 多种语言）数据。LLaMa 3 的语料库还经过文本质量分类器的筛选，而这个分类器的训练数据由 LLaMa 2 生成。

LLaMa 3 的微调

微调的方法是 SFT、Rejection Sampling、PPO 和 Direct Preference Optimization (DPO) 的组合。

LLaMa 3 的后续

最大的 LLaMa 模型参数超过 400B，但还在训练中。他们还会陆续发布多个具有新功能的模型，包括多模态、多语言、更长的上下文窗口等，训练完 LLaMa 3 后，他们还会发一篇详细的研究论文（而不是 blog）。
