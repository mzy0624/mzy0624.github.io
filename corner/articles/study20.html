解决了什么问题

大模型在面对复杂的代码生成需求时性能不好，而人类通常会将复杂问题分解为简单的子问题然后一步一步解决，所以作者提出了 Self-Planning 方法，该方法包含两个阶段：Planning（规划）和 Implementation（实现）。在 Planning 阶段中，大模型从自然语言需求出发，结合少量的 prompts，规划出简洁的步骤；而在 Implementation 阶段，大模型在前面生成的步骤的指导下逐步生成代码。

Planning

规划阶段中，大模型基于 few-shot prompting 将问题分解为更简洁的子问题，其中 prompt $C$ 设计了 $k$ 个相连的样例： $$C\triangleq\langle x_1^e\cdot y_1^e\rangle\ \|\ \langle x_2^e\cdot y_2^e\rangle\ \|\ \cdots\ \|\ \langle x_k^e\cdot y_k^e\rangle$$ 其中 $\langle x_i^e\cdot y_i^e\rangle$ 由样例问题 $x_i^e$ 和它的规划方案 $y^e_i$ 组成，且方案 $y^e_i=\{q_{ij}\}_{j=1}^n$ 由若干个子问题组成。在推理阶段，用于测试的问题 $x$ 会被直接拼接在 prompt 后面然后丢到大模型中，大模型随后输出规划 $y$。

Implementation

将问题 $x$ 和其规划 $y$ 拼接后输入到大模型中，大模型输出最终代码 $z$。

形式化

Planning 和 Implementation 阶段可以形式化为： \begin{align*} P(z\mid x,C)=&\ \sum_{\hat{y}}P(z\mid\hat{y},x,C)\cdot P(\hat{y}\mid x,C)\\ \propto&\ P(z\mid y,x,C)\cdot P(y\mid x,C) \end{align*} 其中 $\hat{y}$ 是任何可能的规划，且 $y$ 代表大模型生成的规划（文章使用概率最高的那一个）。由于条件独立性假设，可以简化 $P(z\mid y,x,C)=P(z\mid y,x)$，于是： $$P(z\mid x,C)\triangleq P(z\mid y,x)\cdot P(y\mid x,C)$$ 前一项代表 Implementation 阶段，后一项代表 Planning 阶段。

Crafting prompts for Self-Planning

作者准备了 $k$ 个样例问题，并且按照以下准则为其制定了规划：

  1. 规划以编号列表的形式组织，列表中的每一项代表一个步骤；
  2. 每一个步骤都代表一个单一、易于实现的子问题，这些子问题被表述为以动词开头的祈使句，重点关注每一步所需的动作；
  3. 步骤应简明扼要，避免过于详细的实施细节；
  4. 规划按照步骤顺序执行，但对于更复杂的结构，规划可以包含分支（包括循环）。
